## 一、设计理念

### 1.1 平台定位
一站式AI学习平台。学习者完成全部课程后应具备：
- **理论能力**：理解核心算法的数学原理，能解释"为什么这样做"
- **动手能力**：能独立完成数据加载→特征工程→建模→评估→部署的完整流程
- **决策能力**：面对新问题时能判断用什么方法、为什么用、什么时候不用

### 1.2 核心原则
1. **知其然+知其所以然**：每章双重验收——能解释原理+能动手实现
2. **每章一个聚焦点**：学完能做一件事、能解释一个原理、能做一个判断
3. **理论驱动实践**：先理解为什么，再学习怎么做，最后判断何时用
4. **从真实问题出发**：每个概念从一个真实场景引入，而非直接给定义

### 1.3 章节结构标准（v2.0）

每章必须包含以下模块：

| 模块 | 作用 | 要求 |
|------|------|------|
| 🎯学习目标 | 三重验收 | 能解释(理解)+能做(动手)+能判断(决策) |
| 动机层 | 为什么需要 | 真实场景痛点 |
| 原理层 | 是什么+为什么 | 直觉类比→数学形式化→代码验证 |
| ❓关键问题 | 为什么不那样做 | 每章至少1个，回答设计选择的理由 |
| 边界层 | 什么时候失效 | 明确适用范围和局限 |
| 实践层 | 怎么做 | 最小示例→完整案例 |
| 误区层 | 避坑 | 表格+最大陷阱(机制解释) |
| 📝练习 | 验证学习效果 | ⭐原理解释 ⭐⭐代码实现 ⭐⭐⭐场景判断 |
| ✅检查点 | 自检 | 与学习目标一一对应 |

### 1.4 练习题三层设计

| 层次 | 考查维度 | 题型示例 |
|------|---------|---------|
| ⭐ 原理解释 | 知其所以然 | "为什么用X不用Y？" "解释Z的几何含义" |
| ⭐⭐ 代码实现 | 知其然 | "用X完成Y任务" "实现Z算法" |
| ⭐⭐⭐ 场景判断 | 知何时用 | "这个场景该用X还是Y？为什么？" "X在什么条件下会失效？" |

### 1.5 "❓关键问题"模块设计原则

这是本课程区别于普通教程的核心模块，专门回答"为什么不那样做"：

1. 每章至少1个关键问题
2. 问题必须是学习者真实会困惑的（不是人造的）
3. 回答要包含：直觉解释+数学证明/代码验证

**示例**：

| 章节 | 关键问题 |
|------|---------|
| 逻辑回归 | 为什么用sigmoid不用阶跃函数？（答：阶跃不可导，无法梯度下降） |
| 交叉熵损失 | 为什么分类用交叉熵不用MSE？（答：MSE+sigmoid导致梯度消失） |
| Random Forest | 为什么随机选特征而不是用全部特征？（答：去相关性，让树有差异） |
| CNN | 为什么用卷积不用全连接？（答：参数共享+平移不变性+局部感知） |
| Transformer | 为什么用Self-Attention不用RNN？（答：并行化+长距离依赖） |
| LoRA | 为什么冻结原权重只训低秩矩阵？（答：过拟合风险低+显存省+可插拔） |

## 九、内容生成规范

### 9.1 每章内容生成Checklist

- [ ] 学习目标包含理解+动手+决策三层
- [ ] 有真实场景的动机段（不是"本章将介绍…"）
- [ ] 有直觉类比（抽象概念必须有生活类比）
- [ ] 有❓关键问题（为什么不那样做？）
- [ ] 有边界层（什么时候失效？）
- [ ] 代码有逐步解析（不能只贴代码不解释）
- [ ] 练习覆盖三层（⭐原理 ⭐⭐代码 ⭐⭐⭐场景）
- [ ] 检查点与学习目标一一对应

### 9.2 代码规范

- Python 3.11+
- 优先用NumPy/sklearn/PyTorch实现
- 代码必须可运行（包含import和数据）
- 变量命名语义化（不用x1/x2/a/b）
- 注释用英文，解释用中文

### 9.3 内容来源标注

- 公式/定理：标注出处教材
- 代码示例：标注改编来源或"原创"
- 数据集：标注来源和许可证
- 图片/图表：标注来源或"原创"

## 十一、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 拆章后章节太多，学习者感到压力 | 学习动力下降 | 按周分组展示，每周末有里程碑 |
| 内容生成量大（110章） | 质量可能不均 | Phase 3试点验证，建立质量标准 |
| 前端展示密集章节体验差 | 学习体验下降 | 折叠/分组/进度条优化 |
| 参考资料版权问题 | 法律风险 | 标注来源，代码示例原创或标注改编 |
| 内容过时（尤其LLM领域） | 教学质量下降 | 设计版本化内容，定期更新机制 |
