# AILP 章节内容模板规范 v1.0

> 每一章的内容必须严格遵循此模板结构，确保学习者能"知其然，知其所以然"。

---

## 一、设计原则

1. **为什么先于怎么做**：每个概念先解释"为什么需要它"，再展示"怎么用它"
2. **直觉先于形式**：用生活类比建立直觉，再用数学/代码精确表达
3. **做中学**：每3-5分钟阅读量后必须有一个动手环节
4. **闭环验证**：章末必须有练习+参考答案，学习者能自检

---

## 三、各模块字数参考

| 模块 | 目标字数 | 说明 |
|------|---------|------|
| 学习目标 | 50-100字 | 精炼，3-5条 |
| 动机层 | 100-200字 | 不啰嗦，直击痛点 |
| 理解层 | 200-400字 | 类比+定义 |
| 应用层 | 300-600字 | 含代码+解析 |
| 误区层 | 100-200字 | 表格+重点陷阱 |
| 课后练习 | 200-400字 | 2-3题 |
| 检查点 | 50-100字 | 与学习目标一一对应 |
| **总计** | **1000-2000字** | 不含代码 |

> 一章总内容（含代码）控制在 **3000-6000字** 为宜，对应 **15-30分钟** 学习时长。

## 四、6个Phase的特殊侧重点

| Phase | 核心变化 | 内容侧重点 |
|-------|---------|-----------|
| Phase 1 Python | 思维转换 | Java→Python对比，"为什么Python这么做"比"怎么做"重要 |
| Phase 2 数学 | 直觉建立 | 公式前必须有直觉解释，代码验证直觉 |
| Phase 3 ML | 范式理解 | 先讲"什么场景用什么方法"，再讲方法本身 |
| Phase 4 DL | 架构演进 | 从动机出发：为什么需要CNN/RNN/Attention，不只是结构描述 |
| Phase 5 LLM | 涌现理解 | 从预训练→微调→推理的因果链，理解"为什么大模型能工作" |
| Phase 6 工程 | 权衡决策 | 每个技术选型都要讲trade-off，没有银弹 |

---

## 五、质量检查清单

每章完成后，对照以下清单自检：

- [ ] 学习目标是否全部可检验？（避免"了解""知道"）
- [ ] 是否有"为什么"动机段？（不直接给方案）
- [ ] 是否有直觉类比？（抽象概念必须有生活类比）
- [ ] 代码是否有逐步解析？（不能只贴代码不解释）
- [ ] 是否有常见误区？（至少2个）
- [ ] 练习是否覆盖所有学习目标？（目标-练习映射）
- [ ] 练习是否有参考答案？（用`<details>`折叠）
- [ ] 检查点与学习目标是否一一对应？
- [ ] 总字数是否在3000-6000范围内？

## 六、模板变量说明

在JSON种子数据中，章节内容存储在 `content` 字段，使用Markdown格式。

Phase 1/2 的真相源是 `app/data/phase{1,2}/index.json`。
Phase 3-6 的真相源是数据库（`create_only`行为）。

修改内容时：
- Phase 1/2 → 修改JSON文件 + upsert到DB
- Phase 3-6 → 直接修改DB（后续需同步回种子文件）
