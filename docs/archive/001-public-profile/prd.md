# PRD: 公开能力主页

> 版本：V1.0
> 日期：2026-05-23
> 作者：PO Agent
> 流程级别：Standard

---

## 背景

AILP 是一个 AI 能力验证平台。用户在学习 AI 课程、完成实验、获得认证后，目前这些能力数据仅停留在平台内部，无法对外展示。V4 PRD 将 AILP 重新定位为「AI 能力的可信验证平台——以实训为入口，以验证为价值，以招聘为自然结果」。

当前痛点：
- **学习者**：在平台上投入了大量时间学习、实验、获得认证，但这些成果无法在简历或招聘平台上展示。传统的简历描述「熟练掌握 Python/机器学习」缺乏可信度，无法与 AILP 平台上的真实数据关联。
- **雇主**：无法高效验证候选人的 AI 能力。简历上的技能陈述难以量化，面试筛选成本高。
- **平台**：学习完成后用户流失，缺乏自然的社交传播和招聘生态的切入点。

公开能力主页是连接「学习验证」与「招聘结果」的关键桥梁——它让用户在平台上的每一点进步都成为可展示、可验证的能力凭证。

---

## 目标

1. **为学习者提供可分享的能力名片**：每个用户拥有一个可公开访问的个人主页 URL（`ailp.com/p/{username}`），一键分享到简历、领英、招聘平台。
2. **建立可验证的能力信任链**：主页数据来源于平台真实的学习记录（完成的实验、获得的认证），而非用户自行填写，确保数据的可信度。
3. **为招聘生态奠定基础**：公开主页是后续雇主搜索/筛选功能的前置依赖，本次先实现「可展示」，为「可搜索」打下数据基础。

---

## 目标用户

| 用户角色 | 特征 | 核心诉求 |
|---------|------|---------|
| 学习者（主页拥有者） | 已在 AILP 完成一定课程/实验/认证，正在求职或希望展示 AI 能力 | 希望有一个专业的能力主页来替代或补充传统简历中的「技能」部分；希望控制哪些数据对外可见 |
| 雇主/招聘方（主页访问者） | HR、技术负责人、猎头，在筛选 AI 岗位候选人 | 希望快速了解候选人的真实 AI 能力水平，看到可验证的数据而非主观陈述 |
| 未注册浏览者 | 通过分享链接访问主页的任何人 | 了解 AILP 平台价值，可能转化为注册用户 |

## 核心功能点

### F1: 公开能力主页页面

每个用户拥有一个独立的公开主页，展示 AI 能力全景。访问者无需登录即可查看。

- 页面 URL 格式为 `/p/{username}`，每个用户拥有唯一标识
- 页面包含：用户基本信息（头像、昵称、一句话简介）、技能雷达图、已完成的实验列表、已获得的认证列表
- 技能雷达图从用户的实际数据中聚合生成（如 "Python 编程"、"机器学习"、"深度学习"、"LLM 应用"、"AI 工程化" 等维度）
- 技能雷达各项能力值根据用户完成对应课程/实验的进度和成绩自动计算
- 页面底部展示「数据由 AILP 平台验证」标识及平台入口链接，增强可信度并为平台引流

### F2: 隐私控制——我的公开主页设置

用户在个人设置中可精细控制每个能力维度的公开/隐藏状态。默认全部隐藏，用户主动开启后才对外可见。

- 提供「一键开启全部」和「一键隐藏全部」的快捷操作入口
- 可独立控制以下维度的可见性：
  - 技能雷达图（开/关）
  - 已完成的实验列表（开/关）
  - 已获得的认证列表（开/关）
  - 个人基本信息（开/关——关闭后主页仅显示用户名，不显示头像和简介）
- **规则**：用户未主动开启的主页，任何人访问时显示「该用户尚未公开能力主页」；用户隐藏的维度，主页上不展示对应模块
- **绝不做未经授权的数据暴露**：任何用户数据在未被用户显式开启公开前，外界无法通过任何方式访问

### F3: 主页分享功能

用户可便捷地将公开主页分享到外部平台。

- 提供「复制链接」按钮，一键复制主页 URL
- 预览模式：用户在设置页可实时预览自己的公开主页效果（所见即所得），确认无误后再分享
- 页面支持 Open Graph 标签，分享到社交媒体（微信、领英、Twitter）时显示富媒体卡片（包含用户昵称和技能摘要）

### F4: 主页管理入口

在用户已登录的个人中心/设置页面提供公开主页的管理入口。

- 入口按钮文案：「我的公开主页」
- 未开启时显示引导提示：「开启你的能力主页，让雇主发现你」，配有一键开启按钮
- 已开启后显示主页链接、开启/关闭开关、各模块隐私设置、预览按钮

---

## 不做的事（Non-Goals）

- **雇主端搜索/筛选功能**：本次不实现雇主通过条件搜索候选人的功能，这属于后续 Sprint 的招聘生态功能
- **用户排名/排行榜**：不在主页上展示「全站排名第 X 名」之类的竞争性数据
- **自定义主页样式/主题**：所有用户使用统一的主页布局和样式，不支持个性化皮肤或自定义 CSS
- **主页访问统计**：不实现访问量计数、访问来源分析等数据分析功能
- **主页内容的手动编辑**：主页上展示的数据全部来源于平台记录，用户不可手动添加或修改（仅能控制显示/隐藏）
- **社交互动功能**：不包含评论、点赞、关注等功能
- **多语言支持**：本次仅支持中文

## 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|:---:|------|
| 公开主页开启率 | ≥30% 的活跃用户在功能上线后 4 周内开启 | 统计开启公开主页的用户数 / 活跃用户总数 |
| 主页分享率 | ≥15% 的开启用户产生至少 1 次分享行为 | 统计点击「复制链接」或分享按钮的用户数 / 开启用户数 |
| 主页访问量 | 上线 4 周内，公开主页累计被非登录用户访问 ≥500 次 | 统计 `/p/{username}` 路径的匿名访问 PV |
| 从主页到注册的转化率 | ≥5% 的匿名访问者注册成为用户 | 从主页点击「在 AILP 学习」入口并完成注册的人数 / 主页匿名 UV |

## UI / UX 设计

### 用户旅程

```
学习者视角：
  个人中心 → 点击「我的公开主页」→ 查看引导/当前设置
    → [未开启] 看到引导卡片 → 一键开启 → 进入设置页
    → [已开启] 进入设置页 → 调整各维度开关 → 实时预览 → 复制链接 → 分享到外部

访问者视角：
  收到链接 → 打开公开主页 → 查看技能雷达 + 实验 + 认证 → 底部平台入口 →（可选）进入 AILP 注册
```

### 设计说明

- 主页整体风格简洁专业，减少装饰元素，突出数据内容
- 技能雷达图作为页面视觉重心，直观展示能力全貌
- 采用卡片式布局展示实验和认证列表，列表过长时默认折叠（显示前 3-5 条），提供「查看全部」展开按钮
- 页面底部统一放置 AILP 品牌标识和数据验证声明，既是信任背书也是平台引流
- 移动端适配：雷达图在小屏幕下改为垂直条形图或简化展示；卡片式列表自然适配单列布局
