## 验收场景

> 每个场景使用 Given-When-Then 格式。覆盖正常路径 + 异常路径 + 边界条件。

### AC1: 访问者查看已开启的公开主页（全部维度可见）

- **Given** 用户 "zhangsan" 已注册 AILP，拥有头像、昵称「张三」、一句话简介「专注 CV 方向的 AI 工程师」，完成了 8 个实验，获得了 2 个认证（Python L2、深度学习 L3）。该用户已在设置中开启公开主页，并将「个人基本信息」「技能雷达图」「已完成实验」「已获得认证」四个维度全部设为可见。
- **When** 访问者（无需登录）通过浏览器打开 `ailp.com/p/zhangsan`
- **Then** 页面展示：用户头像、昵称「张三」、一句话简介「专注 CV 方向的 AI 工程师」；技能雷达图（雷达图或移动端等效展示），展示各维度当前分数；已完成实验列表（8 个），默认显示前 5 条并提供「展开全部」按钮；已获得认证列表（2 个），展示认证名称、等级、获得时间和验证链接（如有）。页面底部展示「✅ 数据由 AILP 平台验证」标识及「在 AILP 上学习 AI 课程」入口链接。页面整体加载完成时间不超过 2 秒。

---

### AC2: 访问者查看已开启的主页（部分维度隐藏）

- **Given** 用户 "lisi" 已开启公开主页，但在设置中将「个人基本信息」和「技能雷达图」设为可见，「已完成实验」和「已获得认证」设为隐藏。该用户昵称为「李四」，有 5 个实验和 1 个认证。
- **When** 访问者通过浏览器打开 `ailp.com/p/lisi`
- **Then** 页面展示用户头像、昵称「李四」和一句话简介，展示技能雷达图。已完成实验列表和已获得认证列表不出现于页面中。页面布局不因此出现空白占位区域（隐藏模块对应区域完全不渲染）。

---

### AC3: 用户首次开启公开主页

- **Given** 用户 "wangwu" 已登录 AILP，此前从未配置过公开主页（全部维度处于默认隐藏状态）。该用户有头像、昵称及少量实验记录。
- **When** 用户在个人中心点击「我的公开主页」入口
- **Then** 页面展示引导卡片，显示文案「开启你的能力主页，让雇主发现你」，并提供一键开启按钮。用户点击一键开启后，公开主页状态变为「已开启」，四个维度的可见性自动全部置为「开」，用户立即可以看到自己的主页链接 `ailp.com/p/wangwu` 和「复制链接」按钮。同时页面提示「你的能力主页已开启，可在下方调整可见性设置」。

---

### AC4: 用户逐步调整可见性设置

- **Given** 用户 "zhaoliu" 已开启公开主页，当前全部维度可见。该用户完成设置调整后准备分享主页。
- **When** 用户在设置页将「个人基本信息」关闭，然后点击「预览我的主页」
- **Then** 预览页面在新标签页打开，展示 `ailp.com/p/zhaoliu` 的实际效果。页面上仅显示用户名（不显示头像和一句话简介），技能雷达图、实验列表和认证列表仍正常展示。预览页面与访问者实际看到的页面完全一致（所见即所得）。用户关闭预览标签页回到设置页，点击「复制链接」按钮，系统复制主页 URL 到剪贴板，并显示 toast 提示「链接已复制到剪贴板」。

---

### AC5: 用户一键隐藏全部维度后访问主页（所有维度隐藏）

- **Given** 用户 "sunqi" 已开启公开主页，在设置页点击「一键隐藏全部」，将四个维度全部关闭。注意：公开主页整体状态仍为「已开启」——用户没有关闭主页，只是隐藏了所有内容维度。
- **When** 访问者打开 `ailp.com/p/sunqi`
- **Then** 页面正常加载，仅展示用户名「sunqi」和页面底部 AILP 品牌标识及平台入口。头像、一句话简介、技能雷达图、实验列表、认证列表均不展示。页面不显示「该用户尚未公开能力主页」的提示（因为主页已开启）。

---

### AC6: 访问不存在的用户名

- **Given** 系统中不存在用户名为 "nonexistent999" 的用户
- **When** 访问者打开 `ailp.com/p/nonexistent999`
- **Then** 页面展示错误信息，明确提示「该用户不存在」。页面不展示任何用户数据区域，仅保留 AILP 导航栏（精简版）和底部品牌标识。页面返回合适的状态码以告知浏览器该页面不存在，且不会被搜索引擎索引。

---

### AC7: 访问未开启公开主页的用户

- **Given** 用户 "zhouba" 已注册 AILP，但从未在设置中开启公开主页（或曾经开启后又将其关闭）
- **When** 访问者打开 `ailp.com/p/zhouba`
- **Then** 页面展示提示「该用户尚未公开能力主页」。页面不展示该用户的任何个人信息、技能数据、实验或认证数据。页面底部保留 AILP 品牌标识和「在 AILP 上学习 AI 课程」入口链接。

---

### AC8: 用户没有任何实验和认证时开启主页

- **Given** 用户 "wujiu" 刚刚注册 AILP，尚未完成任何实验、未获得任何认证。技能雷达各维度分数均为 0 或初始值。该用户仍决定开启公开主页。
- **When** 访问者打开 `ailp.com/p/wujiu`
- **Then** 技能雷达图展示各维度初始状态（分数为 0 或初始值），而非隐藏或显示错误。已完成实验区域显示空状态提示「暂无实验记录」。已获得认证区域显示空状态提示「暂无认证记录」。这些空状态提示文案为中性口吻，不引导用户采取任何行动（访问者不是用户本人）。

---

### AC9: 用户拥有大量实验数据时主页列表展示

- **Given** 用户 "dataking" 已完成 200 个实验，获得 15 个认证。已开启公开主页且全部维度可见。
- **When** 访问者打开 `ailp.com/p/dataking`
- **Then** 实验列表默认展示前 5 条记录，底部显示「展开全部（200 个实验）」按钮。展开后所有实验记录按完成时间倒序排列（最近完成的在前），页面滚动流畅。认证列表展示全部 15 条认证（认证数量通常较少，不需要折叠）。实验列表即使展开全部，页面整体加载时间仍不超过 3 秒。

---

### AC10: 分享链接在社交媒体展示富媒体卡片

- **Given** 用户 "zhangsan" 已开启公开主页，昵称「张三」，一句话简介「专注 CV 方向的 AI 工程师」，技能雷达中「深度学习」维度分数最高
- **When** 访问者将 `ailp.com/p/zhangsan` 链接粘贴到微信、领英或 Twitter 的输入框中
- **Then** 社交媒体自动拉取并展示富媒体预览卡片，包含：标题「张三的能力主页 - AILP」、描述「专注 CV 方向的 AI 工程师」或技能摘要（如「深度学习 L3 · Python L2」）、以及 AILP 平台默认缩略图。卡片信息与用户当前公开主页的内容一致。

---

### AC11: 用户关闭公开主页后已分享的链接失效

- **Given** 用户 "zhangsan" 之前已开启公开主页并分享了链接。现在该用户在设置页将公开主页状态从「已开启」切换为「未开启」。
- **When** 持有旧链接的访问者再次打开 `ailp.com/p/zhangsan`
- **Then** 页面展示「该用户尚未公开能力主页」提示，不再展示任何个人数据。行为与 AC7 一致。

---

### AC12: 并发访问同一公开主页

- **Given** 用户 "zhangsan" 的公开主页已开启且数据完整
- **When** 100 个访问者几乎同时打开 `ailp.com/p/zhangsan`
- **Then** 所有访问者均能在 3 秒内看到完整的主页内容。页面数据一致（所有访问者看到相同的技能分数、实验列表和认证列表）。无访问者遇到超时、崩溃或数据错乱。

## 业务规则

| 规则编号 | 规则描述 | 涉及场景 |
|:---:|------|:---:|
| BR1 | **隐私默认隐藏**：用户注册后，公开主页默认处于「未开启」状态，所有维度默认不可见。任何数据在用户显式开启前，外界无法通过任何方式访问。 | AC3, AC7 |
| BR2 | **数据来源不可编辑**：公开主页上所有展示数据（技能分数、实验列表、认证列表）均来源于用户在平台上的真实学习记录。用户不可手动修改或增删这些数据，仅能控制其是否对外展示。 | AC1, AC2 |
| BR3 | **技能分数动态计算**：技能雷达图的各维度分数由系统根据用户完成的实验和课程数据自动聚合计算，非固定值。用户完成新实验后，对应维度分数自动更新，公开主页同步反映最新分数。 | AC1, AC8 |
| BR4 | **维度独立控制**：四个可见性维度（个人基本信息、技能雷达图、已完成实验、已获得认证）互相独立，用户可以任意组合开启或关闭。关闭某个维度仅影响该模块在主页上的展示，不影响其他维度。 | AC2, AC4, AC5 |
| BR5 | **主页状态与维度状态分离**：「公开主页已开启/未开启」是一个总开关。总开关关闭时，无论维度设置如何，所有数据均不可见。总开关开启但某维度关闭时，仅该维度不可见。总开关从关闭变为开启时，所有维度默认置为「开」。 | AC3, AC5, AC11 |
| BR6 | **所见即所得**：用户在设置页的「预览我的主页」所看到的页面效果，与外部访问者访问主页看到的效果完全一致，包括隐藏模块的处理、空状态展示等。 | AC4 |
| BR7 | **链接持久性**：公开主页 URL（`/p/{username}`）基于用户名生成，同一用户始终拥有同一 URL。即使主页被关闭后重新开启，URL 不变。 | AC11 |
| BR8 | **访问无需登录**：公开主页对所有访问者开放，无需注册或登录 AILP 即可查看已开启的主页内容。 | AC1, AC2, AC5, AC8 |
| BR9 | **已删除用户的主页不可访问**：当某个用户账号被平台删除后，其之前可能存在的公开主页链接立即失效，访问时返回「该用户不存在」。 | AC6 |
| BR10 | **认证验证链接可跳转**：认证记录中如包含外部验证链接，访问者点击后可在新标签页打开，不离开当前主页。验证链接由平台在颁发认证时生成，用户不可修改。 | AC1 |

## 边界条件

| 边界 | 条件 | 预期行为 | 涉及场景 |
|------|------|---------|:---:|
| 零实验零认证 | 用户注册后尚未完成任何实验或获得任何认证 | 实验列表和认证列表展示空状态提示「暂无实验记录」「暂无认证记录」。技能雷达图展示各维度初始值（0 或基线分数），而非空白图表 | AC8 |
| 大量实验数据 | 用户完成 200+ 个实验 | 实验列表默认折叠，仅展示前 5 条；提供「展开全部」按钮，展开后按完成时间倒序排列，页面滚动流畅 | AC9 |
| 大量认证数据 | 用户获得 50+ 个认证 | 认证列表一次性全部展示（认证数量远小于实验数量，无需折叠）。若未来认证数量膨胀，可考虑引入分页展示 | AC9 |
| 用户名包含特殊字符 | 用户名包含 `.`、`-`、`_` 等 URL 合法字符 | 主页 URL 正常工作，访问者能正常访问 | — |
| 用户名过长 | 用户名超过 30 个字符 | 主页页面布局正常展示用户名，不出现溢出或截断（UI 层面做文本省略处理） | — |
| 一句话简介过长 | 用户设置的一句话简介超过 200 个字符 | 主页展示时截断至 200 字符并加省略号「…」，完整内容在鼠标悬停时以 tooltip 展示 | — |
| 用户头像缺失 | 用户未上传头像 | 展示平台默认头像占位图，不影响主页其他内容的展示 | — |
| 并发访问 | 同一主页同时被大量访问者访问 | 所有访问者均能在 3 秒内看到完整页面，数据一致，无超时或崩溃 | AC12 |
| 用户实时修改可见性 | 访问者正在查看主页时，用户修改了可见性设置 | 访问者当前已加载的页面不实时变化（无需推送）。访问者刷新页面后，看到最新设置下的内容 | — |
| 移动端小屏 | 访问者使用手机浏览器打开主页（屏幕宽度 < 768px） | 技能雷达图切换为垂直条形图或等效简化展示（保证可读性）。卡片列表自然适配单列布局。所有文字和按钮正常可读、可点击 | — |
| OG 标签缓存 | 用户修改简介或关闭主页后，社交媒体仍缓存旧卡片 | 社交媒体平台的缓存刷新不由 AILP 控制。AILP 仅保证每次请求返回正确的 OG 标签，实际刷新频率取决于各平台策略 | AC10 |

## 错误场景与用户提示文案

| 错误场景 | 触发条件 | 用户看到的提示文案 |
|---------|---------|-----------------|
| 用户不存在 | 访问者输入的用户名在系统中不存在，或对应用户账号已被删除 | 「该用户不存在」 |
| 主页未开启 | 访问的用户存在但未开启公开主页，或已将主页关闭 | 「该用户尚未公开能力主页」 |
| 页面不存在（通用 404） | 访问者输入了无效的页面路径（非 `/p/{username}` 格式的路径） | 「页面未找到」 |
| 暂无实验记录 | 用户开启了实验列表可见，但尚未完成任何实验 | 「暂无实验记录」 |
| 暂无认证记录 | 用户开启了认证列表可见，但尚未获得任何认证 | 「暂无认证记录」 |
| 链接复制成功 | 用户点击「复制链接」按钮，系统成功复制主页 URL 到剪贴板 | 「链接已复制到剪贴板」 |
| 链接复制失败 | 用户点击「复制链接」按钮，但浏览器不支持剪贴板 API 或因权限问题复制失败 | 「复制失败，请手动复制链接」 |
| 首次开启成功 | 用户从引导卡片点击一键开启后，主页成功激活 | 「你的能力主页已开启，可在下方调整可见性设置」 |
| 主页已关闭 | 用户在设置页将主页状态从「已开启」切换为「未开启」后 | 「你的能力主页已关闭，他人将无法访问」 |

## 关键非功能需求

| 需求类型 | 要求 | 衡量方式 |
|---------|------|:---:|
| **性能 — 首屏加载** | 公开主页页面首次完整加载（含技能雷达图渲染）时间不超过 2 秒，在网络条件良好的情况下（4G/WiFi） | 使用浏览器开发者工具 Performance 面板测量 `/p/{username}` 从导航开始到主要内容渲染完成的时间 |
| **性能 — 大量数据** | 用户拥有 200+ 个实验时，展开全部实验列表后页面滚动帧率不低于 30fps，展开操作响应时间不超过 500ms | 使用 Performance 面板测量展开操作和滚动表现 |
| **性能 — 并发访问** | 同一主页在 100 个并发请求下，所有请求在 3 秒内完成响应 | 压测工具模拟并发，测量 P95 响应时间 |
| **安全 — 未授权数据不可见** | 用户未显式开启的维度和未开启主页的用户，其任何个人数据不可被外部访问者通过任何方式获取（包括直接 URL 访问、爬虫、API 探测等） | 安全测试：以未登录身份尝试访问未开启主页的用户数据，验证所有路径均返回「该用户尚未公开能力主页」或「该用户不存在」，不泄露任何数据片段 |
| **安全 — 数据来源防篡改** | 用户在主页上看到的数据必须与平台记录严格一致，不可被人为修改（用户无法通过设置页或任何前端手段修改实验列表、认证列表、技能分数） | 功能测试：对比用户在个人中心看到的实验/认证/技能数据与公开主页上展示的数据，确保完全一致 |
| **可用性 — 移动端适配** | 公开主页在所有主流移动浏览器（iOS Safari、Android Chrome、微信内置浏览器）上正常展示，内容可读、链接可点击、雷达图有等效简化展示 | 在 iPhone SE（375px 宽）和 Android 中端设备（360px 宽）上手动验收 |
| **可用性 — 渐进增强** | 在禁用 JavaScript 的浏览器中，主页应展示基本的文本信息（昵称、实验名称、认证名称），不出现完全空白页。交互功能（如展开列表、雷达图动画）可降级 | 禁用浏览器 JavaScript 后手动访问主页验收 |
| **可用性 — 浏览器兼容** | 支持 Chrome/Firefox/Safari/Edge 最近两个大版本 | 在目标浏览器上手动验收每个 AC 场景 |
| **SEO — 搜索引擎可索引** | 已开启的公开主页应被搜索引擎索引，页面包含用户昵称、技能摘要等结构化信息。未开启的主页和不存在用户的主页不应被索引 | 检查页面 `<meta name="robots">` 标签及 Open Graph 标签的正确性 |
| **可维护性 — 设置即时生效** | 用户在设置页修改可见性后，修改立即生效（无需审核、无需延迟）。访问者刷新页面即可看到最新设置 | 修改可见性后立即在新浏览器或无痕窗口访问主页验证 |
