## 一、产品概述

### 1.1 产品定义

AI学习平台是一个完整的在线教育平台，提供从Python基础到AI团队管理的全链路学习体验。该平台整合了课程学习、在线实验、能力认证等核心功能，旨在培养具备AI全栈能力的专业人才。

### 1.2 核心价值主张

**For Learners (学习者)**:
- 📚 系统化学习路径：从入门到专家的完整课程体系
- 💻 边学边练：浏览器内直接编写和运行Python代码
- 🏆 能力认证：完成学习后获得能力认证证书

**For Organizations (企业)**:
- 👥 团队培训：批量管理学员学习进度
- 📊 数据分析：追踪学习效果和掌握程度
- 🎯 定向培养：定制化课程内容

### 1.3 产品定位

| 维度 | 说明 |
|------|------|
| 产品类型 | B2C/B2B 在线教育平台 |
| 核心功能 | 课程学习 + 在线实验 + 能力认证 |
| 目标市场 | AI技术培训、企业内部培训 |
| 竞争优势 | 内置完整课程 + 安全代码执行环境 |

### 3.2 认证模块

#### 3.2.1 用户注册

| 需求项 | 描述 |
|--------|------|
| 功能名称 | 用户注册 |
| 功能描述 | 新用户通过填写邮箱、用户名、密码进行注册 |
| 触发方式 | POST `/api/auth/register` |
| 请求格式 | JSON |
| 请求参数 | `email` (string, 必填), `username` (string, 必填), `password` (string, 必填) |
| 返回结果 | UserResponse |
| 业务规则 | - 邮箱唯一性校验<br>- 用户名唯一性校验<br>- 密码强度校验（长度≥8） |
| 错误处理 | - 400: 邮箱已注册<br>- 400: 用户名已存在 |

#### 3.2.2 用户登录

| 需求项 | 描述 |
|--------|------|
| 功能名称 | 用户登录 |
| 功能描述 | 已注册用户登录系统获取访问令牌 |
| 触发方式 | POST `/api/auth/login` |
| 请求格式 | `application/x-www-form-urlencoded` |
| 请求参数 | `username` (string, 必填，值为邮箱), `password` (string, 必填) |
| 返回结果 | Token (access_token, token_type) |
| 业务规则 | - 验证邮箱和密码<br>- 返回JWT令牌<br>- 令牌有效期15分钟 |
| 错误处理 | - 401: 邮箱或密码错误 |
| 注意事项 | 使用OAuth2PasswordRequestForm，非JSON格式 |

#### 3.2.3 获取当前用户

| 需求项 | 描述 |
|--------|------|
| 功能名称 | 获取当前用户信息 |
| 功能描述 | 获取已认证用户的个人信息 |
| 触发方式 | GET `/api/auth/me` |
| 认证方式 | Bearer Token |
| 返回结果 | UserResponse |
| 错误处理 | - 401: 无效认证凭证 |

### 3.4 实验模块

#### 3.4.1 代码执行

| 需求项 | 描述 |
|--------|------|
| 功能名称 | 代码执行 |
| 功能描述 | 在沙箱中执行用户提交的Python代码 |
| 触发方式 | POST `/api/code/execute` |
| 认证方式 | Bearer Token |
| 请求格式 | JSON |
| 请求参数 | code (string, 必填), timeout (int, 可选，默认30秒) |
| 返回结果 | CodeExecutionResponse |
| 业务规则 | - 超时自动终止<br>- 内存限制256MB<br>- 危险代码拦截 |
| 错误处理 | - 执行超时<br>- 内存超限<br>- 危险操作拦截 |
| 安全限制 | 禁止执行: subprocess, eval, exec, os.system, open, 文件I/O等 |

#### 3.4.2 实验提交

| 需求项 | 描述 |
|--------|------|
| 功能名称 | 实验提交 |

| 触发方式 | POST `/api/labs/{lab_id}/submit` |
| 认证方式 | Bearer Token |
| 路径参数 | lab_id (int, 必填) |
| 请求参数 | code (string, 必填) |
| 返回结果 | 提交结果 + submission_id |
| 业务规则 | - 执行代码<br>- 保存提交记录<br>- 返回执行结果 |

### 4.5 安全设计

#### 4.5.1 代码执行安全

| 安全措施 | 描述 |
|----------|------|
| 容器隔离 | 每次执行在独立Docker容器 |
| 资源限制 | CPU: 0.5核, 内存: 256MB, 时间: 30秒 |
| 网络隔离 | 禁止外网访问 |

| 文件系统限制 | 只读挂载，无文件写入权限 |

| 安全措施 | 描述 |
|----------|------|
| 密码加密 | bcrypt |
| Token过期 | 15分钟 |
| Token算法 | HS256 |
| 错误信息 | 通用错误信息（防信息泄露） |
